# ag_ui.client

The Agent User Interaction Protocol Client SDK provides agent connectivity options for Dart applications. This package delivers flexible connection methods to AG-UI compatible agent implementations with full type safety and reactive programming support.

```bash
dart pub add ag_ui
```

## AgUiClient

`AgUiClient` is the main class for connecting to AG-UI compatible servers. It handles HTTP communication, SSE streaming, and binary protocol encoding/decoding.

### Key Features

- **Server-Sent Events (SSE)**: Real-time event streaming with automatic reconnection
- **Binary Protocol**: Efficient data encoding using the AG-UI binary format
- **Error Recovery**: Built-in retry logic with exponential backoff
- **Cancellation Support**: Clean stream cancellation and resource cleanup
- **Type Safety**: Fully typed events and responses

### Configuration

Configure the client with server details and optional parameters:

```dart
final client = AgUiClient(
  config: AgUiClientConfig(
    baseUrl: 'http://localhost:8000',
    defaultHeaders: {'Authorization': 'Bearer token'},
    requestTimeout: Duration(seconds: 30),
    maxRetries: 3,
  ),
);
```

`baseUrl` is the AG-UI server base address. `runAgent()` then accepts either:

- an endpoint path such as `agent` or `agentic_chat`
- an absolute URL such as `https://api.example.com/agentic_chat`

### Core Methods

#### runAgent

Executes an agent with the provided input and streams events:

```dart
Stream<BaseEvent> runAgent(
  String endpoint,
  SimpleRunAgentInput input, {
  CancelToken? cancelToken,
})
```

### Usage Examples

#### Basic Agent Execution

```dart
final input = SimpleRunAgentInput(
  messages: [
    UserMessage(id: 'msg_1', content: 'What is the weather today?'),
  ],
);

await for (final event in client.runAgent('weather-agent', input)) {
  print('Received: ${event.type}');
}
```

The example above sends the request to `baseUrl/weather-agent`. For example, with
`baseUrl: 'http://localhost:8000'`, the final request URL becomes
`http://localhost:8000/weather-agent`.

You can also pass a full URL:

```dart
await for (final event in client.runAgent(
  'https://api.example.com/weather-agent',
  input,
)) {
  print('Received: ${event.type}');
}
```

#### With Error Handling

```dart
try {
  await for (final event in client.runAgent('agent', input)) {
    // Process events
  }
} on ValidationError catch (e) {
  print('Validation error: ${e.message}');
} on TransportError catch (e) {
  print('Transport error: ${e.message}');
}
```

#### Cancellation

```dart
final cancelToken = CancelToken();
final stream = client.runAgent(
  'long-running-agent',
  input,
  cancelToken: cancelToken,
);

// Listen with cancellation option
final subscription = stream.listen((event) {
  // Process events
});

// Cancel when needed
cancelToken.cancel();
await subscription.cancel();
```

## AgUiClientConfig

Configuration object for client initialization:

### Properties

- `baseUrl` (String): Base URL of the AG-UI server
- `defaultHeaders` (Map<String, String>): Default headers for all requests
- `requestTimeout` (Duration): Request timeout duration
- `connectionTimeout` (Duration): SSE connection idle timeout
- `backoffStrategy` (BackoffStrategy): Retry backoff strategy
- `maxRetries` (int): Maximum retry attempts
- `withCredentials` (bool): Whether credentials should be included

## Error Handling

The client provides comprehensive error handling with specific exception types:

### Exception Types

- `ValidationError`: Input validation issues
- `TransportError`: HTTP/SSE transport failures
- `TimeoutError`: Request timeout errors
- `CancellationError`: Request cancellation

### Error Recovery

The client retries using the configured backoff strategy:

```dart
final config = AgUiClientConfig(
  baseUrl: 'http://localhost:8000',
  maxRetries: 5,
  backoffStrategy: ExponentialBackoff(
    initialDelay: Duration(milliseconds: 500),
  ),
);
```

## SSE Client

The underlying SSE client handles Server-Sent Events with:

- Automatic reconnection on connection loss
- Exponential backoff for retry attempts
- Event ID tracking for resumption
- Keep-alive ping/pong support

## Best Practices

1. **Resource Management**: Always close clients when done:
   ```dart
   await client.close();
   ```

2. **Error Handling**: Implement comprehensive error handling:
   ```dart
   stream.handleError((error) {
     // Log and recover
   });
   ```

3. **Cancellation**: Cancel long-running operations when appropriate:
   ```dart
   cancelToken.cancel();
   ```

4. **Headers**: Use headers for authentication and tracking:
   ```dart
   final client = AgUiClient(
     config: AgUiClientConfig(
       baseUrl: 'http://localhost:8000',
       defaultHeaders: {'X-Request-ID': 'unique-id'},
     ),
   );
   ```
