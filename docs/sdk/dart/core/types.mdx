# Core Types

The AG-UI Dart SDK exposes immutable protocol models for runs, messages, tools, context, interrupts, and multimodal user input.

## RunAgentInput

`RunAgentInput` is the canonical protocol model used inside the core SDK.

```dart
class RunAgentInput extends AGUIModel {
  final String threadId;
  final String runId;
  final String? parentRunId;
  final Object? state;
  final List<Message> messages;
  final List<Tool> tools;
  final List<Context> context;
  final Object? forwardedProps;
  final List<ResumeEntry>? resume;

  const RunAgentInput({
    required this.threadId,
    required this.runId,
    this.parentRunId,
    this.state,
    required this.messages,
    required this.tools,
    required this.context,
    this.forwardedProps,
    this.resume,
  });
}
```

### Example

```dart
final input = RunAgentInput(
  threadId: 'thread_123',
  runId: 'run_456',
  messages: [
    UserMessage(id: 'msg_1', content: 'Hello!'),
  ],
  tools: [
    Tool(
      name: 'get_weather',
      description: 'Get current weather',
      parameters: {
        'type': 'object',
        'properties': {
          'location': {'type': 'string'},
        },
      },
    ),
  ],
  context: const [
    Context(description: 'tenant', value: 'acme'),
  ],
  state: {'mode': 'chat'},
);
```

## SimpleRunAgentInput

`SimpleRunAgentInput` is the convenience model used by `AgUiClient.runAgent()`. It keeps most fields optional and serializes missing collections as empty arrays.

```dart
class SimpleRunAgentInput {
  final String? threadId;
  final String? runId;
  final String? parentRunId;
  final List<Message>? messages;
  final List<Tool>? tools;
  final List<Context>? context;
  final Object? state;
  final Map<String, dynamic>? config;
  final Map<String, dynamic>? metadata;
  final Object? forwardedProps;
  final List<ResumeEntry>? resume;

  const SimpleRunAgentInput({
    this.threadId,
    this.runId,
    this.parentRunId,
    this.messages,
    this.tools,
    this.context,
    this.state,
    this.config,
    this.metadata,
    this.forwardedProps,
    this.resume,
  });
}
```

### Example

```dart
final input = SimpleRunAgentInput(
  messages: [
    UserMessage(id: 'msg_1', content: 'Hello!'),
  ],
  context: const [
    Context(description: 'userId', value: 'user123'),
    Context(description: 'sessionId', value: 'session456'),
  ],
  metadata: {'source': 'mobile-app'},
);
```

## Message Types

Messages represent conversation state across user, assistant, tool, activity, and reasoning roles.

### UserMessage

User content can be either a plain string or a list of multimodal content parts.

```dart
class UserMessage extends Message {
  final Object content;

  const UserMessage({
    required super.id,
    required this.content,
    super.name,
    super.encryptedValue,
  });
}
```

### AssistantMessage

Assistant messages may contain text, tool calls, or both.

```dart
class AssistantMessage extends Message {
  final String? content;
  final List<ToolCall>? toolCalls;

  const AssistantMessage({
    required super.id,
    this.content,
    super.name,
    super.encryptedValue,
    this.toolCalls,
  });
}
```

### ToolMessage

Tool messages carry the output of a tool invocation.

```dart
class ToolMessage extends Message {
  final String content;
  final String toolCallId;
  final String? error;

  const ToolMessage({
    required super.id,
    required this.content,
    required this.toolCallId,
    this.error,
    super.encryptedValue,
  });
}
```

### Other Message Roles

- `DeveloperMessage`
- `SystemMessage`
- `ActivityMessage`
- `ReasoningMessage`

### Example Conversation

```dart
final conversation = [
  const SystemMessage(
    id: 'sys_1',
    content: 'You are a helpful assistant.',
  ),
  UserMessage(
    id: 'user_1',
    content: [
      const TextInputContent(text: 'What is the weather?'),
      ImageInputContent(
        source: const InputContentUrlSource(
          value: 'https://example.com/map.png',
          mimeType: 'image/png',
        ),
      ),
    ],
  ),
  AssistantMessage(
    id: 'asst_1',
    content: 'Let me check the weather for you.',
    toolCalls: const [
      ToolCall(
        id: 'call_1',
        function: FunctionCall(
          name: 'get_weather',
          arguments: '{"location":"San Francisco"}',
        ),
      ),
    ],
  ),
  const ToolMessage(
    id: 'tool_1',
    toolCallId: 'call_1',
    content: '{"temperature":68,"conditions":"sunny"}',
  ),
];
```

## Multimodal Input Content

The SDK models multimodal user content with `InputContent` variants:

- `TextInputContent`
- `ImageInputContent`
- `AudioInputContent`
- `VideoInputContent`
- `DocumentInputContent`
- `BinaryInputContent`

These source-backed variants use either `InputContentDataSource` or `InputContentUrlSource`.

```dart
final multimodalMessage = UserMessage(
  id: 'user_2',
  content: [
    const TextInputContent(text: 'Summarize this file'),
    DocumentInputContent(
      source: const InputContentUrlSource(
        value: 'https://example.com/report.pdf',
        mimeType: 'application/pdf',
      ),
    ),
  ],
);
```

## Context

Context entries are simple description/value pairs.

```dart
class Context extends AGUIModel {
  final String description;
  final String value;

  const Context({
    required this.description,
    required this.value,
  });
}
```

### Example

```dart
const context = [
  Context(description: 'tenant', value: 'acme'),
  Context(description: 'locale', value: 'en-US'),
];
```

## Tool

`Tool` describes callable capabilities exposed to the agent.

```dart
class Tool extends AGUIModel {
  final String name;
  final String description;
  final Object? parameters;
  final Map<String, dynamic>? metadata;

  const Tool({
    required this.name,
    required this.description,
    this.parameters,
    this.metadata,
  });
}
```

### ToolCall And FunctionCall

Assistant tool invocations are represented with `ToolCall` and `FunctionCall`.

```dart
class FunctionCall extends AGUIModel {
  final String name;
  final String arguments;

  const FunctionCall({
    required this.name,
    required this.arguments,
  });
}

class ToolCall extends AGUIModel {
  final String id;
  final String type;
  final FunctionCall function;
  final String? encryptedValue;

  const ToolCall({
    required this.id,
    this.type = 'function',
    required this.function,
    this.encryptedValue,
  });
}
```

### ToolResult

`ToolResult` is a client-side convenience model for posting tool execution results.

```dart
class ToolResult extends AGUIModel {
  final String toolCallId;
  final String content;
  final String? error;
  final Map<String, dynamic>? metadata;

  const ToolResult({
    required this.toolCallId,
    required this.content,
    this.error,
    this.metadata,
  });
}
```

## Interrupts And Resume

The core protocol also models human-in-the-loop interruptions and resume payloads.

```dart
class Interrupt extends AGUIModel {
  final String id;
  final String reason;
  final String? message;
  final String? toolCallId;
  final Map<String, dynamic>? responseSchema;
  final String? expiresAt;
  final Map<String, dynamic>? metadata;

  const Interrupt({
    required this.id,
    required this.reason,
    this.message,
    this.toolCallId,
    this.responseSchema,
    this.expiresAt,
    this.metadata,
  });
}

class ResumeEntry extends AGUIModel {
  final String interruptId;
  final ResumeStatus status;
  final Object? payload;

  const ResumeEntry({
    required this.interruptId,
    required this.status,
    this.payload,
  });
}
```

## State

Agent state is intentionally modeled as `Object?` so snapshots and deltas can carry arbitrary JSON-like structures.

```dart
Object? state = {
  'conversation': {'messageCount': 1},
  'user': {'satisfaction': 'high'},
};
```

## Serialization

All protocol models implement `toJson()` and support `fromJson(...)` constructors or factories.

```dart
final json = input.toJson();
final tool = Tool.fromJson({
  'name': 'get_weather',
  'description': 'Get current weather',
});
```
